Полное написание type: entity в прототипе

- type: entity
  id: MyItem
  parent: [ parent1, parent2 ]
  name: "my item"
  description: "my best item"
  suffix: "IT IS MY ITEM"
  abstract: false
  noSpawn: false
  components:
  - type: ...

ФОРМАТИРОВАНИЕ ПРОТОТИПОВ ОБЯЗАТЕЛЬНО!

Разбор всех типов переменных

id

Уникальный идентификатор, или имя, прототипа в коде. Используется для идентификации прототипа внутри игры.
Пример использования: id: MyItem
Пример использования: id: ClothingBackpackDuffelSyndicateEliteHardsuitBundle
Пример использования: id: WeponPistol
Пример использования: id: ClothingBackpackBundle

parent

Родитель прототипа, определяющий отношения наследования компонентов. Может быть одним родителем или массивом родителей, указанным в квадратных скобках.
Пример указания одного родителя: parent: parent1
Пример указания одного родителя: parent: WeponPistol
Пример указания одного родителя: parent: ClothingBackpackDuffelSyndicateEliteHardsuitBundle
Пример указания нескольких родителей: parent: [ parent1, parent2 ]
Пример указания нескольких родителей: parent: [ parent214, parent222, parent237, parent2158 ]

name

Имя предмета. Может быть любым текстом. Если имя содержит специфичные символы, используйте кавычки.
Пример использования: name: "my item"
Пример использования: name: "not my item"
Пример использования: name: item

description

Описание предмета при его осмотре. Работает аналогично имени предмета.
Пример использования: description: "my best item"
Пример использования: description: "NEKO FUCK U"
Пример использования: description: Ok

suffix

Суффикс предмета, отображаемый при спавне через меню. Работает аналогично имени и описанию.
Пример использования: suffix: "my best suffix"
Пример использования: suffix: suffix
Пример использования: suffix: weapon

abstract

Определяет, является ли прототип абстрактным. Если true, прототип не может быть заспавнен напрямую. В спавн меню не отображается.
.
Пример использования: abstract: false
Пример использования: abstract: true

noSpawn

Определяет, отображается ли прототип в спавн меню. Если true, прототип не отображается, но его можно заспавнить через команду.
Пример использования: noSpawn: false
Пример использования: noSpawn: true

components

Блок, содержащий компоненты для энтити. Компоненты добавляют функциональность прототипу. Список компонентов будет предоставлен позже на отдельной странице.
